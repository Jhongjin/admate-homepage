# AdMate Media Planner Brief v1

작성일: 2026-04-30  
문서 상태: 초안 v1  
대상: 미디어플래너 / 캠페인 운영 실무자  
작성 목적: AdMate가 실제 미디어플래너의 업무를 어떻게 줄이고, 실수를 어떻게 막고, 캠페인 운영 품질을 어떻게 높이는지 실무 관점에서 설명하기 위한 문서. 향후 미디어플래너 대상 발표 슬라이드의 원본 자료로 활용한다.

---

## 1. 한 줄 정의

AdMate는 미디어플래너를 대체하는 시스템이 아니라, 반복 확인·정책 검색·세팅 검수·캡처·운영 이상 감지를 대신 처리해 플래너가 전략과 판단에 집중하도록 돕는 **AI 업무 파트너**다.

```text
AdMate
기획부터 운영까지 미디어플래너의 반복 업무를 줄여주는
AI Agent 기반 광고 운영 자동화 플랫폼
```

---

## 2. 미디어플래너가 매일 겪는 문제

미디어플래너의 업무는 단순히 캠페인을 세팅하고 운영하는 일이 아니다.

하나의 캠페인을 준비하고 운영하기 위해 다음과 같은 반복 업무가 계속 발생한다.

| 업무 상황 | 현재 어려움 |
|---|---|
| 정책 확인 | 매체별 정책과 가이드가 흩어져 있어 직접 검색해야 함 |
| 캠페인 세팅 | 미디어믹스와 실제 매체 세팅값이 맞는지 수작업 확인 필요 |
| 예산/기간/타겟 검수 | 작은 숫자 단위 오류가 큰 사고로 이어질 수 있음 |
| 랜딩 URL 확인 | 오타, 404, 잘못된 페이지 연결 가능성 존재 |
| 운영 중 모니터링 | 예산 소진, CPC/CTR 변화, 캠페인 상태를 사람이 수시로 확인해야 함 |
| 캡처/보고 | 광고 게재 화면 캡처와 보고서용 증빙 이미지를 반복 생성해야 함 |
| 다음 제안 준비 | 과거 성과와 업종별 기준을 찾고 비교하는 데 시간이 걸림 |

이런 업무는 중요하지만, 대부분 전략적 판단보다 반복 확인에 가깝다.

AdMate는 이 반복 확인 업무를 줄이고, 플래너가 더 중요한 판단에 집중하도록 돕는다.

### 슬라이드 전환 포인트

- 슬라이드 제목: **미디어플래너의 반복 업무는 어디서 발생하는가**
- 핵심 메시지: 정책 확인, 세팅 검수, 캡처, 운영 모니터링은 중요하지만 반복 리소스가 크다.
- 시각화 아이디어: 업무 카드 6개를 Before 형태로 배열

## 3. AdMate가 제공하는 캠페인 운영 사이클

AdMate는 캠페인의 전 과정을 하나의 흐름으로 연결한다.

```text
기획
→ 정책 확인
→ 캠페인 세팅 검수
→ 집행 시작
→ 운영 모니터링
→ 캡처/보고
→ 다음 캠페인 예측
```

각 단계에서 사용하는 플랫폼은 다음과 같다.

| 단계 | 플랫폼 | 플래너가 얻는 도움 |
|---|---|---|
| 기획 | AdMate Foresight | 업종/목표/예산 기준 예상 성과 확인 |
| 정책 확인 | AdMate Compass | 매체 정책과 가이드를 빠르게 질문 |
| 사전 검수 | AdMate Sentinel | 미디어믹스와 실제 세팅값 오류 자동 확인 |
| 운영 감지 | AdMate Sentinel | 예산/KPI/상태 이상을 자동 감지 |
| 캡처/보고 | AdMate Lens | 광고 게재 화면과 증빙 이미지 자동 생성 |
| 학습 | AdMate Agent Core | 운영자 판단이 다음 기준에 반영 |

AdMate는 플래너가 각 업무를 따로 처리하지 않도록, 캠페인 흐름에 따라 필요한 기능을 연결한다.

### 슬라이드 전환 포인트

- 슬라이드 제목: **AdMate 캠페인 운영 사이클**
- 핵심 메시지: AdMate는 기획부터 운영, 보고, 다음 기획까지 연결한다.
- 시각화 아이디어: 원형 사이클 또는 왼쪽에서 오른쪽으로 흐르는 프로세스 다이어그램

## 4. AdMate Compass: 정책 확인 시간을 줄이다

### 4.1 현재 문제

광고 정책은 매체마다 다르고, 자주 바뀐다.

Meta, Google, Naver, Kakao, X의 정책과 가이드를 직접 검색하고 확인하는 데 시간이 걸린다.

특히 광고 소재, 랜딩 페이지, 업종 제한, 표현 규정, 이미지/영상 규정 등은 캠페인 집행 전 반드시 확인해야 하지만, 매번 담당자의 경험과 검색에 의존하기 쉽다.

### 4.2 AdMate Compass가 하는 일

AdMate Compass는 광고 플랫폼 정책과 가이드의 방향을 잡아주는 Policy Intelligence Agent다.

플래너는 다음과 같이 질문할 수 있다.

```text
이 소재 Meta 정책상 문제 있을까?
건기식 광고에서 주의해야 할 문구 알려줘.
Google Ads에서 랜딩 페이지 관련 정책 확인해줘.
Naver 검색광고에서 병의원 업종 제한 사항 알려줘.
```

Compass는 매체 공식 가이드와 내부 지식 기반으로 답변하고, 가능한 경우 근거를 함께 제공한다.

### 4.3 기대 효과

- 정책 확인 시간 단축
- 특정 담당자 지식 의존도 감소
- 광고주 문의 대응 속도 향상
- 정책 리스크 사전 감소

### 플래너 입장에서의 변화

```text
Before
매체별 도움말 검색 → 문서 탐색 → 담당자에게 재확인 → 답변 정리

After
AdMate에 질문 → 공식 가이드 기반 답변 확인 → 필요한 경우 근거 링크 확인
```

### 슬라이드 전환 포인트

- 슬라이드 제목: **Compass: 정책 확인을 더 빠르게**
- 핵심 메시지: 흩어진 매체 정책을 하나의 질문 경험으로 통합한다.
- 시각화 아이디어: 질문 입력창 + 정책 답변 카드

## 5. AdMate Sentinel: 캠페인 사고를 줄이다

AdMate Sentinel은 캠페인 시작 전과 시작 후를 모두 감시한다.

```text
Pre-launch Validation
= 시작 전 세팅 검수

Live Monitoring
= 시작 후 실시간 운영 감지
```

## 5.1 시작 전: 세팅 오류 검수

### 현재 문제

캠페인 세팅은 작은 실수가 큰 사고로 이어질 수 있다.

예를 들어:

- 예산 단위 오류
- 기간 오류
- 타겟 오류
- 랜딩 URL 오류
- 캠페인 상태 오류
- 미디어믹스와 실제 세팅값 불일치

이런 오류를 사람이 하나씩 비교하는 방식은 시간이 많이 걸리고, 캠페인이 많아질수록 부담이 커진다.

### Sentinel이 하는 일

Sentinel은 미디어믹스/엑셀의 캠페인 세팅 정보와 실제 매체 광고 플랫폼 내 세팅값을 비교한다.

확인 항목 예시:

- 예산
- 기간
- 타겟
- 캠페인 상태
- 랜딩 URL
- 캠페인명/광고세트명
- 구매 유형
- 캠페인 목표

### 기대 효과

- 캠페인 시작 전 오류 발견
- 수작업 검수 시간 감소
- 사고 가능성 감소
- 검수 이력과 timestamp 자동 기록

## 5.2 시작 후: 실시간 모니터링과 알림

### 현재 문제

캠페인 시작 후에도 플래너는 계속 상태를 확인해야 한다.

- 예산이 너무 빨리 소진되는지
- 예산이 거의 쓰이지 않는지
- CTR이 급락했는지
- CPC/CPM이 급등했는지
- 캠페인 상태가 꺼졌는지
- 알림이 너무 많이 오지는 않는지

### Sentinel Live Monitoring이 하는 일

Sentinel은 캠페인 운영 중 주요 이상 신호를 감지하고 Slack/Email로 알려준다.

또한 운영자는 알림에 대해 다음과 같이 대응할 수 있다.

- 확인 완료
- 1시간 보류
- 오늘 중지
- 알림 종료
- 알림 재개
- 검토 필요

이 대응 이력은 AdMate Agent Core에 기록되고, Hermes가 이후 유사 캠페인 판단 기준을 고도화하는 데 활용한다.

### 기대 효과

- 수시 확인 부담 감소
- 이상 상황 조기 발견
- 알림 피로도 통제
- 운영자 판단 기준 축적

### 플래너 입장에서의 변화

```text
Before
직접 매체 관리자 접속 → 수치 확인 → 이상 여부 판단 → 팀원 공유

After
이상 신호 자동 감지 → Slack 알림 확인 → 버튼으로 대응 → 이력 자동 기록
```

### 슬라이드 전환 포인트

- 슬라이드 제목: **Sentinel: 시작 전 오류와 시작 후 이상을 모두 감지**
- 핵심 메시지: 캠페인 사고를 줄이고 운영 확인 부담을 낮춘다.
- 시각화 아이디어: 사전 검수와 실시간 모니터링을 좌우 비교

## 6. AdMate Lens: 캡처와 증빙 업무를 줄이다

### 6.1 현재 문제

광고 운영과 보고 과정에서는 광고 게재 화면, 상품, 지면, 소재 이미지 등 다양한 캡처가 필요하다.

하지만 캡처 업무는 반복적이고 시간이 많이 든다.

- 직접 페이지 접속
- 화면 구성 확인
- 광고 위치 맞추기
- 이미지 저장
- 보고서에 삽입
- 매체/지면별 형태 맞춤

### 6.2 AdMate Lens가 하는 일

AdMate Lens는 광고 게재 화면과 보고서용 증빙 이미지를 자동 생성한다.

예시:

- YouTube 광고 게재 화면
- 모바일 인피드 광고 화면
- 네이티브 광고 UI
- 상품/브랜드 페이지 캡처
- 보고서용 고해상도 이미지

### 6.3 기대 효과

- 반복 캡처 시간 감소
- 보고서 준비 속도 향상
- 이미지 품질 표준화
- 캠페인 기록과 캡처 자산 연결

### 플래너 입장에서의 변화

```text
Before
직접 화면 구성 → 캡처 → 편집 → 보고서 삽입

After
캠페인/매체 정보 입력 → 자동 캡처 생성 → 보고서용 이미지 사용
```

### 슬라이드 전환 포인트

- 슬라이드 제목: **Lens: 캡처와 증빙을 자동화**
- 핵심 메시지: 보고서 준비에 들어가는 반복 리소스를 줄인다.
- 시각화 아이디어: 캠페인 입력 → 자동 캡처 → 보고서 이미지

---

## 7. AdMate Foresight: 다음 캠페인을 더 빠르게 예측하다

### 7.1 현재 문제

캠페인 제안이나 기획 단계에서 플래너는 과거 성과와 업종별 기준을 참고해야 한다.

하지만 다음과 같은 어려움이 있다.

- 과거 유사 캠페인을 찾기 어렵다.
- 업종/목표별 평균 CPM, CPC, CTR 기준이 분산되어 있다.
- 시즌이나 시장 상황을 반영하기 어렵다.
- 담당자 경험에 따른 편차가 발생한다.
- 제안 준비 시간이 오래 걸린다.

### 7.2 AdMate Foresight가 하는 일

AdMate Foresight는 과거 광고 데이터와 시장 트렌드를 기반으로 예상 성과를 제공한다.

1차 PoC는 Meta를 우선 대상으로 한다.

예상 제공 지표:

- CPM
- CPC
- CTR
- VTR
- CPV

입력 예시:

- 업종
- 캠페인 목표
- 예산
- 기간
- 국가
- 소재 유형

결과 예시:

```text
유사 업종/목표 기준 예상 CPM
예상 CPC
예상 CTR
예산 대비 예상 클릭/노출
참고할 시즌성 또는 시장 단가 변화
```

### 7.3 기대 효과

- 기획 단계 의사결정 시간 단축
- 업종/목표별 단가 기준 내부 자산화
- 제안 품질 표준화
- 실제 운영 결과와 다음 기획의 연결

### 플래너 입장에서의 변화

```text
Before
과거 사례 검색 → 엑셀 정리 → 담당자 경험 보정 → 예상 효율 산출

After
업종/목표/예산 입력 → 예상 지표 확인 → 제안 방향 수립
```

### 슬라이드 전환 포인트

- 슬라이드 제목: **Foresight: 다음 캠페인의 성과를 미리 예측**
- 핵심 메시지: 과거 데이터 기반으로 기획 단계의 판단 시간을 줄인다.
- 시각화 아이디어: 입력 폼과 예측 결과 카드

## 8. AdMate Agent Core: 내 판단이 다음 기준이 되다

AdMate의 중요한 차별점은 단순 자동화가 아니라 학습이다.

AdMate Agent Core는 Openclaw와 Hermes를 기반으로 한다.

- Openclaw: 조건과 스케줄에 따라 업무를 실행하는 자동화 실행 엔진
- Hermes: 사용자 이벤트와 운영자 판단을 학습해 지식을 축적하는 지능/메모리 엔진

### 플래너에게 중요한 의미

운영자가 알림에 대해 다음과 같이 판단했다고 가정한다.

```text
이 캠페인은 CPC가 20% 올랐지만,
해당 업종/목표에서는 정상 범위로 볼 수 있음
```

이 판단이 기록되면, Hermes는 유사한 상황에서 더 나은 판단 기준을 만들 수 있다.

즉, 플래너의 경험이 개인의 기억으로만 남지 않고 회사의 운영 지식으로 축적된다.

### 기대 효과

- 반복되는 오탐 감소
- 업종/목표별 판단 기준 고도화
- 운영자 경험의 데이터화
- 다음 캠페인 기획과 운영에 반영

### 슬라이드 전환 포인트

- 슬라이드 제목: **Agent Core: 플래너의 판단을 회사의 지식으로**
- 핵심 메시지: AdMate는 사용할수록 운영 판단 기준을 축적한다.
- 시각화 아이디어: 운영자 버튼 클릭 → Hermes 학습 → 다음 알림 개선

## 9. 실제 업무 시나리오

## 시나리오 1. 캠페인 집행 전 정책 확인

```text
상황:
건기식 캠페인 소재 문구가 정책상 가능한지 확인해야 함

기존 방식:
매체 도움말 검색 → 유사 문서 확인 → 담당자 문의 → 광고주 답변 정리

AdMate 사용:
Compass에 질문 → 공식 가이드 기반 답변 확인 → 주의 문구 정리

효과:
정책 확인 시간 단축, 답변 근거 확보
```

## 시나리오 2. 캠페인 시작 전 세팅 검수

```text
상황:
Meta 캠페인 세팅 완료 후 미디어믹스와 실제 세팅값 일치 여부 확인 필요

기존 방식:
엑셀과 매체 관리자 화면을 번갈아 보며 수작업 비교

AdMate 사용:
Sentinel이 예산/기간/타겟/URL/상태 비교
오류 항목만 표시

효과:
검수 시간 감소, 캠페인 사고 예방
```

## 시나리오 3. 운영 중 이상 감지

```text
상황:
캠페인 CPC가 전일 대비 급등

기존 방식:
플래너가 직접 수치 확인 후 이상 여부 판단

AdMate 사용:
Sentinel이 이상 감지 → Slack 알림 → 운영자가 확인/보류/중지 선택

효과:
대응 속도 향상, 이력 자동 기록
```

## 시나리오 4. 광고주 보고용 캡처 준비

```text
상황:
캠페인 보고서에 광고 게재 화면 캡처 필요

기존 방식:
직접 화면 구성 → 캡처 → 편집 → 보고서 삽입

AdMate 사용:
Lens가 매체/상품/지면별 캡처 자동 생성

효과:
캡처 업무 시간 감소, 증빙 이미지 품질 표준화
```

## 시나리오 5. 다음 캠페인 기획

```text
상황:
동일 업종의 다음 캠페인 제안 준비

기존 방식:
과거 사례 검색, 엑셀 정리, 담당자 경험 기반 보정

AdMate 사용:
Foresight가 업종/목표/예산 기준 예상 성과 제공
Hermes가 기존 운영 피드백 참고

효과:
제안 준비 시간 단축, 예측 기준 표준화
```

## 12. 예상 질문과 답변

### Q1. AdMate가 플래너 업무를 대체하는 건가요?

아니다. AdMate는 반복 확인과 검수, 캡처, 이상 감지를 줄여 플래너가 전략과 판단에 집중하도록 돕는 도구다.

### Q2. 알림이 너무 많이 오면 오히려 피곤하지 않나요?

Sentinel은 알림 보류, 오늘 중지, 알림 종료, 재개 기능을 제공한다. 또한 운영자 피드백을 통해 불필요한 알림을 줄이는 방향으로 고도화할 수 있다.

### Q3. 내 판단이 시스템에 반영되나요?

권한이 있는 사용자의 판단은 Hermes 학습 후보로 기록될 수 있다. 이를 통해 유사 캠페인에서 더 나은 알림 기준과 추천 기준을 만들 수 있다.

### Q4. 정책 답변은 믿을 수 있나요?

Compass는 매체 공식 가이드와 내부 지식 기반으로 답변하는 것을 목표로 한다. 향후 답변 생성 모델과 검증 모델을 분리해 정확도를 높이는 구조로 고도화한다.

### Q5. 모든 캠페인에 바로 적용되나요?

초기에는 일부 매체와 캠페인 유형을 중심으로 PoC와 베타 운영을 진행하고, 검증된 영역부터 확장하는 방식이 적합하다.
